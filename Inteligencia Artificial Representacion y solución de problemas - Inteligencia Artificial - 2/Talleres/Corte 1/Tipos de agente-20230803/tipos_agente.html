El ambiente del laberinto ¶

(Volver al inicio)

Imagine un agente dentro de un laberinto, el cual consiste de una rejilla de doce por doce casillas. El laberinto tiene solo una casilla que sirve de entrada y de salida, que es la casilla $(0,0)$. Observe que en la notacin de las casillas, la primera coordenada representa la posicin en el ejer horizontal y la segunda en el vertical, con origen la esquina inferior izquierda. Esta casilla es la nica forma de salir o entrar al laberinto. Cada rejilla es o bien un pasadizo o bien un muro. El agente puede moverse una casilla en direccin vertical u horizontal (nunca diagonal) siempre y cuando la casilla a la que se mueve no sea un muro o el fin de la rejilla. El problema que debe resolver el agente es encontrar un camino que lo lleve hasta la salida.

In [46]:
lab = Laberinto()
lab.render()

La percepcin del agente consiste en una coleccin de sensores que se encienden cuando detectan un obstculo, organizados de la siguiente manera:

[sensor frontal, sensor izquierdo, sensor derecho, sensor trasero]

Los estados del Laberinto los constituyen los valores de los sensores:

In [47]:
lab = Laberinto()
state = lab.reset()
print('El estado inicial es:')
print(f'[sensor frontal={state[0]}, sensor izquierdo={state[1]}, sensor derecho={state[2]}, sensor trasero={state[3]}]')

El estado inicial es:
[sensor frontal=False, sensor izquierdo=False, sensor derecho=True, sensor trasero=True]

Las acciones posibles del agente son:

  • adelante: el agente avanza una casilla.
  • voltearIzquierda: el agente gira 90 en contra de las manecillas del reloj.
  • voltearDerecha: el agente gira 90 a favor de las manecillas del reloj.

Cada accin del agente tiene un efecto en el entorno, implementado mediante el mtodo step(action) de la clase Laberinto. Este mtodo recibe la accin del agente y devuelve un nuevo estado, una recompensa, y la informacin de si el entorno sigue activo. Un agente dirigido por tabla ¶

(Volver al inicio)

El tipo ms sencillo de un programa de agente es cuando hacemos una asociacin directa entre input y output, en este caso, entre percepcin y accin. Esta asociacin se puede realizar mediante una tabla.

Definimos la siguiente tabla (que hasta ahora slo est definida parcialmente), la cual vincula perceptos con acciones. La tabla implementa la idea de que si el agente percibe que el frente no est bloqueado y el flanco derecho est bloqueado, entonces avanza una casilla:

In [49]:
# (sensor frontal, sensor izquierdo, sensor derecho, sensor trasero)
tabla = {
    # Si el frente no est bloqueado y el flanco derecho est
    # bloqueado, avanzar una casilla
    (False, True, True, True):['adelante'],
    (False, True, True, False):['adelante'],
    (False, False, True, True):['adelante'],
    (False, False, True, False):['adelante'],
}

Note que esta regla tan simple requiere ser expresada mediante cuatro filas de la tabla.

Un agente de reflejo simple ¶

(Volver al inicio)

El proceso de escribir una tabla es bastante dispendioso, pues hay que considerar una gran cantidad de combinaciones de perceptos de los sensores. Un tipo de agente de nivel un poco ms elevado que resuelve esta situacin son los agentes de reflejo simple. El programa de este tipo de agentes est basado en relgas de condicin-accin, las cuales relacionan condiciones sobre los sensores y las acciones. Al considerar condiciones en lugar de combinaciones de perceptos, la escritura es ms eficiente.

A continuacin presentamos una posible implementacin de un agente de reflejo simple para el problema del laberinto, implementado en la clase RuleBased del mdulo agentes.

    # bloqueado, avanzar una casilla.
    'not state[0] and state[2]': ['adelante'],
